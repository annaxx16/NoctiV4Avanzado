## Detalle de cambios

### 1. Cierre por resolución sin slippage
`execution/paper.py::execute_close` acepta `at_resolution: bool`. En un mercado
resuelto el contrato vale exactamente 1.0/0.0; no hay book contra el que cruzar,
así que se omite slippage y el clamp `[0.001, 0.999]`. El exit engine lo activa
en el trigger `t1_outcome_resolved`. Antes una posición ganadora resuelta nunca
cobraba el $1 completo y una perdedora no llegaba a $0.

### 2. Gate time-to-resolution
`risk/engine.py` ahora consulta `Market.end_date` (gate 6.5) y rechaza entradas
en mercados que resuelven en menos de `max_time_to_resolution_hours_floor` horas
(default 2h). Evita entrar justo antes del salto a 0/1.

### 3. Clamp en probability engine
`engine/probability.py::compute_p_fair` clampa a `(0.001, 0.999)`. Sigue siendo
passthrough de la EMA (la calibración bayesiana es trabajo de la v2, GAP-01).

### 4. Aislamiento transaccional del poller
`scheduler/poller.py` envuelve la persistencia de cada snapshot y cada
`evaluate_market` en su propio `SAVEPOINT` (`session.begin_nested()`). Un fallo
de DB en un mercado ya no envenena la transacción ni descarta el tick entero.

### 5. Auth en /admin/*
`require_admin` (header `X-Admin-Token` == `settings.admin_token`), fail-closed:
si `ADMIN_TOKEN` no está configurado, los endpoints admin devuelven 503.

### 6. Batch fetch a Gamma
`polymarket/client.py::get_markets_by_condition_ids` descarga varios mercados por
request (chunks de 50, parámetro `condition_ids` repetido) con fallback
individual. El poller pasa de N round-trips a ~⌈N/50⌉.

### 7. N+1 en position_views
`portfolio/manager.py` hace bulk-prefetch de outcomes y últimos books (2 queries
para todo el set vía `DISTINCT ON`) en vez de ~2 queries por posición. Prioridad
de precio preservada: outcome resuelto → cache Redis → último book en DB.

### 8. Helper compartido
Nuevo `edges/common.py` con `mid()` y `ema()` públicos, usados por
`overreaction.py` y `exit_engine.py`.

### 9. Tests offline
`tests/conftest.py` tolera ausencia de Postgres/Redis: los tests de lógica pura
(sizer, overreaction, slippage, features, leakage, probability) corren sin infra.
Los de integración fallan con error claro si falta la DB.

```bash
# Tests puros (sin infra):
pytest tests/test_sizer.py tests/test_overreaction.py \
       tests/test_paper_execution.py tests/test_probability.py tests/leakage/
```

### 10. Tooling de calidad
`ruff` + `mypy` configurados en `pyproject.toml`; deps en `requirements-dev.txt`.
`ruff check src tests` pasa limpio (96 issues auto-corregidos + ajustes manuales).

```bash
pip install -r requirements-dev.txt
ruff check src tests
mypy src
```
